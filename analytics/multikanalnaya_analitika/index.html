    Мультиканальная аналитика

    Мультиканальная аналитика нужна для правильной оценки эффективности каждого рекламного источника. В CoMagic есть возможность строить отчеты по 8 моделям атрибуции, а также по 2 способам мультиканального анализа.

    Наиболее распространенные модели атрибуции:

    • Последнее взаимодействие - При использовании этой модели для каждого взаимодействия определяется последний источник перехода, без учета истории посещений посетителя.
    • Последнее непрямое взаимодействие - При использовании этой модели для каждого взаимодействия определяется источник предшествующий источнику прямого перехода.
    • Первое взаимодействие - При использовании этой модели для каждого взаимодействия определяется первый источник перехода посетителя, т.е. тот источник который первым привел на сайт вашего посетителя.

    Дополнительно в отчете есть возможность воспользоваться следующими моделями:

    • Линейная - При использовании этой модели каждое взаимодействие разделяется на количество посещений совершенных посетителем до данного взаимодействия, т.е. каждому посещению присваивается равная часть взаимодействия.

    • С учётом давности взаимодействия - При использовании этой модели присваивается большая ценность взаимодействия тем посещениям, которые находятся ближе к моменту взаимодействия. Ценность уменьшается в два раза за каждые 7 дней до взаимодействия. То есть, посещение, которое было за 7 дней до взаимодействия, имеет в два раза меньшую ценность, чем посещение в день взаимодействия. При этом посещение которое было за 14 дней до взаимодействия, имеет в 4 раза меньшую ценность. Если для вашего бизнеса требуется применить модель U-Shape или задать нужные настройки весов воспользуйтесь одной из моделей:

    • С учётом привязки к позиции (40-20-40) - При использовании этой модели каждому первому и последнему посещению присваивается 40% взаимодействия, а оставшиеся 20% взаимодействия равномерно распределяются оставшимся посещениям.

    • С учётом привязки к позиции (20-60-20) - При использовании этой модели каждому первому и последнему посещению присваивается 20% взаимодействия, а оставшиеся 60% взаимодействия равномерно распределяются оставшимся посещениям.
    • С учетом привязки к позиции (10-50-40) - При использовании этой модели каждому первому посещению присваивается 10% взаимодействия, последнему посещению присваивается 40% взаимодействия, а оставшиеся 50% взаимодействия равномерно распределяются оставшимся посещениям.

    Способы мультиканального анализа:

    • Ассоциированные взаимодействия - взаимодействия, которые вспомогательно принес канал.
    • Всего взаимодействий - Сумма уникальных взаимодействий который принес канал как напрямую так и ассоциированно.

    Отчет "Анализ рекламы" по умолчанию строится по наиболее распространенной модели атрибуции "Последнее взаимодействие" - Last Click.

    Для построения отчета по нужной модели атрибуции/способу мультиканального анализа необходимо:

    • Перейти в раздел "Настроить столбцы":
    • Выбрать "Создать столбец" - С фильтром

    • Указать название столбца (обязательно к заполнению). Также при необходимости вы можете указать "Описание столбца".

    • В разделе "Выбор параметра" выбирете метрику, для которой нужно применить модель атрибуции/способ мультиканального анализа:

    • ВСЕ метрики, в которых возможно использовать Модель атрибуции/способы мультиканального анализа:

      • Обращения, количество
      • Звонки, количество
      • Заявки, количество
      • Чаты, количество
      • Цели, количество
      • Сделки, количество
      • Выручка, сумма

    • После выбора метрики становится доступен раздел настройки "Мультиканальный анализ":

    • Из списка нужно выбрать необходимую модель атрибуции/способ мультиканального анализа и нажать кнопку "Создать":
    • Созданный столбец отображается в разделе "Мои столбцы" и доступен для добавления в отчет:
    • В отчете метрики отображаются со значком, указывающим на выбранную модель атрибуции/способу мультиканального анализа:

    Для решения такой задачи в отчете "Анализ рекламы" выводим информация по сумме расходов, посещений, обращений и выигранных сделок по рекламной кампании "Яндекс Директ":

    Если в отчет добавить кол-во ассоциированных обращений и сделок, т.е. посмотреть, сколько Яндекс Директ приносит обращений/сделок в качестве вспомогательного канала привлечения клиентов:

    Здесь по интегрированной рекламной кампании РСЯ видим 2 выигранные сделки. Так, можно раскрыть отчет до объявлений и ключей, чтобы определить работающую рекламу и увеличить на нее бюджет.

    Также при анализе интегрированной рекламной кампании "Поиск РФ" можно увидеть объявления, по которым нет обращений и сделок, но при этом есть расходы. Здесь стоит задуматься об отключении таких групп объявлений, либо корректировки их содержания.
