    Получение списка событий

    Метод get.events
    Описание Получение списка всех событий
    Кому доступен Агент, Клиент

    Параметры запроса

    Название Тип Обязательный Описание
    access_token string да Ключ сессии аутентификации
    user_id number нет Уникальный идентификатор пользователя клиента агента от имени которого делается запрос
    Является обязательным для агента
    Для получения списка пользователей клиента необходимо использовать метод "get.customer_users"
    limit number нет Количество возвращаемых записей. См. раздел "Постраничный вывод"
    offset number нет Сдвиг, определяет с какого номера записи возвращать "limit" записей. По умолчанию "0". См. раздел "Постраничный вывод"
    filter object нет См. раздел "Критерии фильтрации"
    fields array нет См. раздел "Представление возвращаемых данных"
    sort array нет См. раздел "Сортировка данных"

    Параметры ответа

    Название Тип Допустимые значения Фильтрация Сортировка Описание
    id number да да Уникальный идентификатор события
    name string да да Название события
    description string да да Описание события
    site_id number да да Уникальный идентификатор сайта
    site_domain_name string да да Адрес сайта в интернете. Без указания протокола - "http://" или "https://"
    type enum
    • comagic - Системное событие. Не может быть удалено
    • track_event - Пользовательское событие. В настройке используются поля category, action, action, value.
    • ppc_goal - Интегрированные цели Яндекс.Метрики и Google Analytics. Всегда имеют флаг is_goal = true, заполнены поля engine, ext_id, goal_name.
    • link - Переход по ссылке. Используются поля url, в котором хранится регулярное выражение или ссылка для отслеживания.
    • time_on_site - Время на сайте. Используются поля value, в котором хранится значение отслеживаемого времени при котором срабатывает событие в формате hh:mm:ss.
    • time_on_page - Время на конкретной странице сайта. Используются поля value, в котором хранится значение отслеживаемого времени при котором срабатывает событие в формате hh:mm:ss и url, в котором хранится отлеживаемая страница сайта.
    • click - Клик на объект. Используются поля value, в котором хранится значение отслеживаемого элемента и url, в котором хранится отлеживаемая страница сайта. Если urlне задан, значит отслеживаются все страницы.
    • scroll - Действия с прокруткой. Используются поля value, в котором хранится тип скролла (comagic_scroll_start, comagic_scroll_end) или отслеживаемый элемент до которого происходит скролл и url, в котором хранится отлеживаемая страница сайта.
    • segment - Вхождение в сегмент. Используются поля segment_id, в котором хранится идентификатор сегмента при вхождении в который срабатывает событие.
    		 да да Типы событий
    category string да да Категория отслеживаемого события типа track_event
    action string да да Действие отслеживаемого события типа track_event
    label string да да Ярлык отслеживаемого события типа track_event
    value string да да Значение отслеживаемого события типа track_event или время для типа события time_on_site,time_on_page или отслеживаемый элемент для типа события click, scroll
    url string да да URL адрес страницы сайта, привязанный к настраиваемому типу события
    segment_id number да да Идентификатор сегмента при вхождении в которой происходит событие. Используется для событий типа segment
    is_goal boolean
    • true
    • false
    		 да да Является ли событие целью. Для событий типа ppc_goal всегда true
    goal_name string да да Название цели, если у события стоит флаг is_goal = true
    engine enum
    • yandex.metrika
    • google.analytics
    		 да да Внешняя аналитическая система являющаяся источником цели. Для событий типа ppc_goal
    ext_id string да да Внешний идентификатор цели из внешней системы. Для событий типа ppc_goal
    is_active boolean
    • true
    • false
    		 да да Включен ли сбор данной цели из внешней аналитической системы. Для событий типа ppc_goal
    ga_category string да да Категория для передаваемого события в Google Analytics
    ga_action string да да Действие для передаваемого события в Google Analytics
    ga_label string да да Ярлык для передаваемого события в Google Analytics
    ga_value string да да Значение для передаваемого события в Google Analytics
    ym_goal_id string да да Идентификатор цели Яндекс Метрики

    JSON структура запроса

    {
  "jsonrpc":"2.0",
  "id":"number",
  "method":"get.segments",
  "params":{
    "access_token":"string",
    "user_id":"number",
    "offset":"number",
    "limit":"number",
    "filter":{

    },
    "sort":[
      {
        "field":"string",
        "order":"string"
      }
    ],
    "fields":[
      "string"
    ]
  }
}
